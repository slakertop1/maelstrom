## [0.4.1] — 2026-07

Полное независимое ревью всего кода (безопасность и надёжность) + доработка отложенного бэклога. Обратно совместимо.

### Безопасность
- Маскирование URL с query-параметрами (presigned-S3, токены, ключи) в отчётах CLI/HTML и на фронте; редакция SQL-литералов и логов; экспорт CLI-конфига предупреждает о секретах открытым текстом вместо ложного «self-contained».
- gRPC `.proto`: устранены arbitrary-write через имя и path-traversal через импорты (в т.ч. восьмеричные/hex/unicode эскейпы и импортируемые файлы); защита от «proto-бомбы» глубокой вложенностью.
- HTTP: лимит размера ответа (защита от OOM на гигантском теле); редиректы не переносят кастомные секретные заголовки на другой хост и при downgrade https→http; `state.json` создаётся с правами 0600 (Unix, атомарно).

### Добавлено
- **gRPC TLS**: свой CA / клиентский сертификат / mTLS — сквозная поддержка (движок → UI → CLI-экспорт).

### Исправлено
- gRPC-метрики под нагрузкой считали статусы неверно (успешный вызов помечался как сетевая ошибка).
- Assertions: числовые сравнения (`19.9` == `"19.90"`), объекты дают явный mismatch; графики устойчивы к NaN/∞.
- CLI: пустой/нулевой прогон при заданном пороге больше не «проходит» гейт; ошибки записи отчёта логируются; предупреждение, если порог задан для HTTP/gRPC/WS (где он не применялся).
- UI: гонки событий разных прогонов (по поколению), устаревшие ответы медленных запросов, потеря фокуса в TLS-редакторе, двойной старт по двойному клику, фризы на больших телах/логах, поля-пароли для токенов и строк подключения.
- Понятный вывод RPS для цепочек: суммарный помечен как «Σ по шагам», целевой (последний шаг) выведен явно.
- База данных: реальный счётчик строк вместо размера превью.

### Надёжность
- Отмена (Stop) прерывает уже отправленные запросы и загрузку датасетов (таймауты + `CancellationToken`, включая чтение тела ответа); слот нагрузки не залипает при панике таски.
- Стриминг выборок БД без материализации в память (защита от OOM под нагрузкой).
- WebSocket: единый дедлайн round-trip, ограниченная очередь замеров, штатное закрытие соединения.
- OAuth: нарастающая пауза при сбое фонового обновления токена + видимость сбоя в логе GUI.
- Лог нагрузки: файловый I/O вынесен из рабочего потока (не тормозит темп).
- Тесты: +100 (Rust и фронт); нестабильные (флаки) тесты стабилизированы.

## [0.4.0] — 2026-07

### Добавлено
- **Цепочки запросов / потоки (request chaining)** — прогон = набор параллельных
  потоков; поток = шаги по порядку со своим RPS (темп старта цепочек, открытая
  модель). Значение из ответа шага извлекается (`extract`: JSON-путь / заголовок /
  regex) в переменную `{{name}}` и подставляется в следующие шаги. Переменные
  изолированы по итерации (виртуальные пользователи не делят токены). Один шаг =
  одиночная нагрузка; смесь цепочек и одиночных потоков — штатно.
- **Трёхуровневый отчёт** для потоков: overall → поток (завершённых цепочек,
  success-rate, e2e p50/p95/p99, недоданные) → шаги (метрики per-endpoint;
  воронка видна по счётчикам). Панель «🔗 Нагрузка цепочками» на коллекции.
- **CLI:** блок `streams` в конфиге (`targets` стал опциональным); гейт
  `--min-success-rate` (порог доли завершённых цепочек по каждому потоку).
- **Демо в Kubernetes:** пайплайн `k8s-loadtest.yml` — CLI как Job в kind-кластере
  нагружает сервис цепочкой и гейтит (пример для CI/k8s, `deploy/k8s-demo/`).

### Изменено
- Инструмент переехал в публичный репозиторий; CI и релизы бесплатны.

## [0.3.0] — 2026-07

### Добавлено
- **Профили авторизации** — заполните авторизацию (Bearer/Basic/OAuth2) один раз,
  сохраните как профиль и применяйте в любой другой ручке из выпадающего списка.
- **Приватный S3 (AWS SigV4)** — датасеты и наборы файлов по URL умеют подпись
  запроса ключами AWS (access key / secret / session token / регион); в CLI-конфиге
  секреты — через `${VAR}`.
- **Фиксированный RPS** в нагрузке одиночного запроса: заданный целевой RPS
  выдерживается независимо от скорости ответов (открытая модель, как в
  мульти-сценарии и CLI); пустое поле — прежний режим «от VU».
- Счётчик недоданных запросов: если целевой RPS не удержан, недостача видна
  в отчёте, CLI и UI — а не молча занижённый RPS.

### Исправлено
- Правки запроса (в т.ч. авторизация) больше не теряются при переключении на
  другую ручку без нажатия «Сохранить» — автосохранение при переключении.
- Предупреждение «Не заданы переменные» показывалось под окном нагрузки сервиса
  (выглядело как «ничего не происходит») — теперь поверх; плюс неразрешённые
  переменные видны прямо при отметке ручки в сценарии, с подсказкой про
  синтаксис `{{$data.имя.колонка}}`.
- OpenAPI-импорт: схема apiKey-в-заголовке больше не превращается в Bearer.
- Ошибка чтения сохранённого состояния больше не приводит к тихой потере
  коллекций (баннер + защита файла от перезаписи).
- Датасет с URL/S3: не-2xx ответ — ошибка, а не разбор HTML/XML как CSV.
- Гистограмма распределения: устранён двойной счёт на узких диапазонах.

### Изменено
- Подсказка в окружениях — сквозной пример `DB_PASSWORD` от поля до переменной
  в кластере; понятные тултипы у полей нагрузки (две модели нагрузки).

## [0.2.0] — 2026-07

### Добавлено
- gRPC: вызов по `.proto` без protoc (unary + server/client/bidi-стриминг), нагрузка,
  автоподбор корней импортов.
- WebSocket: подключение, отправка/приём, нагрузка.
- Проверки ответа (assertions): статус/время/заголовок/тело/JSON-поле.
- Данные из БД как источник (в приложении и в CLI), наборы файлов (папка/список/URL-S3),
  секреты по окружениям (`${KEY}` в CI-конфиге), кнопка «Тест подключения» у датасета.
- Локализация: английский — базовый язык + переключатель EN/RU.
- Экспорт для CI: диалог с переименованием env-переменных и списком переменных для кластера;
  кнопка «↓ Export for CI» также на вкладке Load одиночного запроса.
- Распространение CLI: бинарники Windows/macOS/Linux + Docker-образ
  `ghcr.io/slakertop1/maelstrom-cli`; примеры GitHub Actions / GitLab CI / Kubernetes;
  живой demo-API (`examples/demo-api`) со Swagger и готовым сценарием нагрузки.
- Сквозное логирование с маскировкой секретов; кнопка «Сообщить об ошибке».

## [0.1.0] — 2026-07

Первый публичный бета-выпуск.

### Возможности
- HTTP-клиент с коллекциями и переменными окружения `{{var}}`.
- Нагрузочное тестирование (VUs / длительность / лимит RPS), живые графики, HTML-отчёт.
- OAuth2/SSO с авто-обновлением токена, mTLS-сертификаты.
- Нагрузка на БД (Postgres / MySQL / SQLite), импорт OpenAPI/Swagger.
- Мульти-эндпоинт «нагрузка сервиса», headless-CLI для CI/Kubernetes + Docker.
