## [0.3.0] — 2026-07

### Добавлено
- **Профили авторизации** — заполните авторизацию (Bearer/Basic/OAuth2) один раз,
  сохраните как профиль и применяйте в любой другой ручке из выпадающего списка.
- **Приватный S3 (AWS SigV4)** — датасеты и наборы файлов по URL умеют подпись
  запроса ключами AWS (access key / secret / session token / регион); в CLI-конфиге
  секреты — через `${VAR}`.
- **Фиксированный RPS** в нагрузке одиночного запроса: заданный целевой RPS
  выдерживается независимо от скорости ответов (открытая модель, как в
  мульти-сценарии и CLI); пустое поле — прежний режим «от VU».
- Счётчик недоданных запросов: если целевой RPS не удержан, недостача видна
  в отчёте, CLI и UI — а не молча занижённый RPS.

### Исправлено
- Правки запроса (в т.ч. авторизация) больше не теряются при переключении на
  другую ручку без нажатия «Сохранить» — автосохранение при переключении.
- Предупреждение «Не заданы переменные» показывалось под окном нагрузки сервиса
  (выглядело как «ничего не происходит») — теперь поверх; плюс неразрешённые
  переменные видны прямо при отметке ручки в сценарии, с подсказкой про
  синтаксис `{{$data.имя.колонка}}`.
- OpenAPI-импорт: схема apiKey-в-заголовке больше не превращается в Bearer.
- Ошибка чтения сохранённого состояния больше не приводит к тихой потере
  коллекций (баннер + защита файла от перезаписи).
- Датасет с URL/S3: не-2xx ответ — ошибка, а не разбор HTML/XML как CSV.
- Гистограмма распределения: устранён двойной счёт на узких диапазонах.

### Изменено
- Подсказка в окружениях — сквозной пример `DB_PASSWORD` от поля до переменной
  в кластере; понятные тултипы у полей нагрузки (две модели нагрузки).

## [0.2.0] — 2026-07

### Добавлено
- gRPC: вызов по `.proto` без protoc (unary + server/client/bidi-стриминг), нагрузка,
  автоподбор корней импортов.
- WebSocket: подключение, отправка/приём, нагрузка.
- Проверки ответа (assertions): статус/время/заголовок/тело/JSON-поле.
- Данные из БД как источник (в приложении и в CLI), наборы файлов (папка/список/URL-S3),
  секреты по окружениям (`${KEY}` в CI-конфиге), кнопка «Тест подключения» у датасета.
- Локализация: английский — базовый язык + переключатель EN/RU.
- Экспорт для CI: диалог с переименованием env-переменных и списком переменных для кластера;
  кнопка «↓ Export for CI» также на вкладке Load одиночного запроса.
- Распространение CLI: бинарники Windows/macOS/Linux + Docker-образ
  `ghcr.io/slakertop1/maelstrom-cli`; примеры GitHub Actions / GitLab CI / Kubernetes;
  живой demo-API (`examples/demo-api`) со Swagger и готовым сценарием нагрузки.
- Сквозное логирование с маскировкой секретов; кнопка «Сообщить об ошибке».

## [0.1.0] — 2026-07

Первый публичный бета-выпуск.

### Возможности
- HTTP-клиент с коллекциями и переменными окружения `{{var}}`.
- Нагрузочное тестирование (VUs / длительность / лимит RPS), живые графики, HTML-отчёт.
- OAuth2/SSO с авто-обновлением токена, mTLS-сертификаты.
- Нагрузка на БД (Postgres / MySQL / SQLite), импорт OpenAPI/Swagger.
- Мульти-эндпоинт «нагрузка сервиса», headless-CLI для CI/Kubernetes + Docker.
